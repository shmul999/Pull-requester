Структура:

avitotask/
-build/ - Docker
-cmd/pull_requester - main.go и миграции
-config/ Конфиг файлы
-internal/ Внутренние пакеты
--domain/ Домены
--infrastructure/ Репозитории, хэндлеры
---http работа с сетью - мидлвейр, хэндлеры, dto, сам сервер
---storage - миграции и бд
-logger логирование(slog) 

Решения:
1)В качестве БД выбрал Postgres
2)В качестве логера всегда использую чаще всего slog - нравится по идиоматике, также сделал для него собственный врапинг ошибок.
3)Docker вынес в директорию build, чтобы не нагружать корень проекта большим кол-вом файлов
4)Также добавил интересную фичу, которую люблю делать в своих проектах - разные окружения для разных конфигураций и автоматическое определение окружения
5)Остальное все согласно заданию

Проблемы:
1)при работе с миграциями, go:embed не находил папку с файлами миграции(migrations), прошло только радикальное решение - переместить все миграции в директорию с файлом main.go
2)Проблема с правами в docker, сервис падал с ошибкой permission denied. Для этого я убрал volume
3)Также была проблема с nil ptr в логере. Для этого я добавил default

4)Не успеваю сделать дополнительные задания, но протестировал все локально. На самом деле еще есть идеи по кешированию через Redis, но надеюсь показать свои умения уже на стажировке =)
